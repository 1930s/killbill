Analytics

Overview

Killbill maintains a reporting schema that is kept current in realtime for Analytics purposes. In contrary to other tables in the system, this set of tables is stable and potentially changes only between major revisions of Killbill.

To distinguish with Killbill objects, the Analytics objects are prefixed with Business, e.g. a Killbill account will have a representation in the Analytics subsystem by a Business Account object. For ease of tracking, Analytics objects contain id references to the Killbill ones but we strongly advise against using them for reporting purposes. Instead, use the external keys as much as possible, as they are tied to your business concepts. For instance, to count the number of accounts in the system, run a query using count(distinct account_key), not count(distinct account_id).

Accounts

The table bac (stands for Business Accounts) maintains information about Killbill accounts. The table doesn't contain historic records (it will contain at most one record per Killbill account).

Column name Column type Description
record_id int(11) unsigned Unique identifier for that record, e.g. 10
account_id char(36) Killbill account id, e.g. b2d69f1e-27cb-4c43-b997-71c28da4a170
account_key varchar(50) External key associated with that account
name varchar(100) Name associated with that account, e.g. Jean DuJardin
created_date bigint(20) Epoch when the account was created, e.g. 1341957285516
updated_date bigint(20) Epoch when the account was last updated, e.g. 1341957519902
balance decimal(10,4) Account balance (takes into account invoices balance and credits), e.g. 254.4500
last_invoice_date bigint(20) Last invoice date for that account, e.g. 1341957522967
total_invoice_balance decimal(10,4) Sum of all invoices balance for that account, e.g. 299.7500
last_payment_status varchar(255) Last payment status, e.g. SUCCESS
payment_method varchar(100) Default payment method (if any), e.g. Paypal
credit_card_type varchar(50) Default card type, if any, associated with that account, e.g. VISA
billing_address_country varchar(50) Billing country for that account, e.g. FR
currency varchar(50) Default account currency, e.g. MXN

Invoices and invoice items

updated_date bigint(20) Epoch when the invoice was last updated (should always be created_date as invoices are immutable), e.g. 1341958418930

currency char(5) Invoice currency, e.g. EUR

amount_paid decimal(10,4) Amount paid, e.g. 0.0000

Overdue status

The table bos (stands for Business Overdue Status) maintains information about Killbill bundles overdue status. The table contains historic status for a given bundle.

Column name Column type Description
record_id int(11) unsigned Unique identifier for that record
bundle_id char(36) Killbill bundle id, e.g. ce3d45ec-30b7-4489-949c-d80808111993
external_key varchar(50) Bundle external key
account_key varchar(50) Account external key
status varchar(50) Overdue status, e.g. PRE_WARNING
start_date bigint(20) Epoch of the overdue status start period, e.g. 1342396800000
end_date bigint(20) Epoch of the overdue status end period, e.g. 1342396800000
