Note: in the current version, there is no authentification mechanism. We provide, however, a way for the client to pass context information through the use of specific HTTP headers:

  • X-Killbill-CreatedBy
  • X-Killbill-Reason
  • X-Killbill-Comment
Each POST/PUT/DELETE needs at least a non null X-Killbill-CreatedBy header; failure to do so will return a 400 (BAD_REQUEST) error code.
