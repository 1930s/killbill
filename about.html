What is Killbill?

Killbill is project aimed at handling online subscription management and its associated billing component. We started this project as part of Ning to replace our existing billing system. We decided to open source the project so others can also benefit from it.

The core of the project is a library that takes as an input a powerful catalog and offers a set of APIs to manage accounts and their subscriptions. The system detects subscriptions events (creation/cancellation/change) to automatically create new invoices and attempt making payments. In addition to the core library, there is also a web server module that offers a set of REST endpoints, an administrative UI, and some client libraries (php, java, ruby, javascript) offering a complete stack.

The payment system is architected as a plugin framework where one can add its own plugin-- for instance a Paypal plugin or a plugin for some specific payment gateways. By default the system comes with a noop plugin that does not do anything.

Quick overview of the Abstractions

A user can create a Subscription by specifying a Plan. The set of existing plans are configured in a catalog, where each plan has a set of Phases. Existing currently supported phases are trials, discounts, evergreen and each phase is defined with some attributes such as a set of prices and a length. Subscriptions have a type; currently supported types are BP (base pan), AO (addon) and standalone. Subscriptions are grouped together in a Bundle to allow specifying business rules across a group of subscriptions. An example is to create one BP along with multiple AOs in a bundle and specify that upon BP cancellation AO should also be cancelled.

Features

Catalog

  • Allow to specify the descriptions of the Plans available in the system.
  • Versioned Catalog, where Plans can be added, old Plans deprecated, prices updated.
  • Allow to specify the business logics associated with cancellation, or change of plans; an example is for instance to specify whether a cancellation should occur immediately triggering some proration logics or occurring at the end of its term (EOT) to avoid any proration. see catalog rules (TODO)
  • Allow to specify Billing alignments to decide which subscriptions should be billed together; an example is to create an invoice at the account level where each subscriptions on this account appear in one invoice.
  • Allow to specify Plan alignments; an example of such alignment allows to define whether changing from one plan to another one will end up restarting from the very first phase of the second plan or not see catalog alignment rules (TODO)
  • Multiple Currencies

Account Management

  • Creation of new accounts
  • Update of account information
  • Deletion of account

Subscription Management

  • Creation of new subscriptions
  • Change of Plan for a given subscription
  • Cancellation of a subscription

Invoicing

  • Automatic invoicing
  • Allows to credit account
  • Generation of custom html invoice through custom template
  • Email invoices optional

Payments

  • Ability to add a payment method
  • Ability to change a payment method
  • Ability to set a default payment method
  • Support for external iFrame for credit card specification
  • Allow to contact any third party payment provider through custom plugins
  • Build-in payment retry in case of system failures
  • Allows external payments
  • Allows full or partial refund
  • Support for chargedbacks

Overdue (Dunning)

  • Credit card failures are retried using configurable policies
  • Custom actions upon payment failures are configurable

Analytics

  • Built-in analytics metrics such as MRR (monthly recurring revenue)
  • Ability to build custom metrics

Requirements

Runtime

  • Java JRE: killbill has been tested against OpenJDK6 (1.6.0_23), OpenJDK7 (1.7.0_147-icedtea) and Oracle JDK7 (Oracle JDK7)
  • Container (optional): killbill has been tested against Jetty 7.5.1 (Servlet ~3.0), Jetty 8.1.3 (Servlet 3.0) and Tomcat 7.0 (Servlet 3.0). JBoss AS 7.1 (Java EE 6, full profile) support is experimental.
  • MySQL database: 5.5 or later recommended
  • Killbill should not have any other system dependency, although it was only tested under Ubuntu 10.0.4, Ubuntu 12.0.4 and Mac OSX 10.6.8

Compile

  • Java SDK: killbill has been compiled and tested against OpenJDK6 (1.6.0_23), OpenJDK7 (1.7.0_147-icedtea) and Oracle JDK7 (Oracle JDK7)
  • Maven: 3.0.3 or later preferred, but 2.2.1 (and later) is also supported
